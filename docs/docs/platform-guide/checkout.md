# Checkout & Payments

The **Checkout Page** handles the final, critical step of the booking flow. It orchestrates price calculations, discount applications, wallet coin deductions, and secure payment processing.

![Checkout Mockup](/img/platform/checkout_mockup.png)

## UI Breakdown

The checkout view (`client/user/src/features/turf/pages/CheckoutPage.jsx`) provides a split-pane or dual-card layout to ensure maximum transparency during payment.

### Booking Summary Card
A static summary of the user's intent:
- Venue Image and Name (`Olympia Arena`).
- Date, Time, and Duration (`2 HOURS`).
- Activity Type (`CRICKET`).

### Financial Breakdown & Payment Options
- **Line Items:** Shows the base booking fee and calculates applicable taxes (`18% Facility Tax`).
- **Coupons/Offers:** An input field to validate and apply promotional codes, instantly reflecting the discount in the total.
- **Wallet Integration:** A toggle switch allowing users to redeem their `Available Coins` against the final total.
- **Call to Action:** A vibrant, high-contrast `PROCEED TO PAY` button that locks the UI and triggers the payment gateway.

## Code Flow & Security

The checkout process relies on backend validation at every step to prevent price tampering.

### 1. Calculation & Order Generation
When the user clicks "Proceed", the frontend sends the selected slots and applied wallet/coupon flags to the server. The backend recalculates the total.

```javascript
const handlePayment = async () => {
  setIsProcessing(true);
  try {
    // 1. Backend calculates final amount and generates an order
    const { data } = await axiosInstance.post("/api/user/bookings/create-order", {
      turfId: turf._id,
      selectedSlots,
      useWalletBalance: useWallet,
      couponCode: appliedCoupon
    });

    if (data.totalAmount === 0) {
      // 100% discount or fully paid via Wallet -> Bypass Razorpay
      handleDirectConfirmation(data.bookingId);
      return;
    }

    launchRazorpay(data.order);
  } catch (error) {
    toast.error(error.response?.data?.message || "Checkout failed");
  } finally {
    setIsProcessing(false);
  }
};
```

### 2. Payment Gateway Launch
If a monetary transaction is required, Razorpay is initialized with the secure `order_id` generated by the backend.

```javascript
const launchRazorpay = (order) => {
  const options = {
    key: import.meta.env.VITE_RAZORPAY_KEY_ID,
    amount: order.amount,
    currency: "INR",
    name: "Kridaz Sports",
    description: "Venue Booking",
    order_id: order.id,
    handler: async (response) => {
      // 3. Signature verification post-payment
      await verifyPaymentSignature(response);
    },
    theme: { color: "#55DEE8" }
  };
  const rzp = new window.Razorpay(options);
  rzp.open();
};
```

### 3. Concurrency & Slot Locking
To handle high demand, Kridaz employs a temporary locking mechanism. When the checkout page loads, the slots are temporarily reserved for a few minutes (preventing double-booking). If the checkout times out or fails, the slots are automatically released back to the public pool.
